# UI Audit
**Date:** 2026-04-28
**Commit:** 66df5c0e44b466f528b5b6a2c841a36a74f144e1

## Summary
- **The biggest problem is a crash bug**: clicking *Start recording* on Live Audio Analyzer calls `setLanguageWarning(null)` ([LiveAnalyzer.jsx:124](src/pages/LiveAnalyzer.jsx#L124)) but the state setter was renamed to `setUnsupportedLanguage`. Every attempt to start the audio analyzer throws `ReferenceError: setLanguageWarning is not defined` and the page is dead.
- The Live Session active screen stacks ~10 simultaneously visible blocks (4 banners + topic + timer + status bar + video/gauge + 6 signal bars + tips + transcript + graph + stop). It is overwhelming and the camera feed has to fight the gauge for the user's eye.
- Two stylesheets are loaded together and contradict each other: [index.css](src/index.css) is the leftover Vite template (light-mode-friendly, sets `#root { width: 1126px; border-inline: 1px solid; text-align: center }`) while [App.css](src/App.css) builds a dark, 900px-wide left-aligned app. The leftover gives every page a gratuitous vertical border and forces all default text to centred.
- The four Home cards split modes into "Live Practice", "Analyze Recording", "Speech Analyzer", "Library" — the first three names overlap semantically and a first-time visitor cannot tell which is video, audio, or which has live scoring.
- "What next" is invisible after a result. The page bottom offers *Back to Library*, *Share*, and a destructive *Discard & re-take* — there is no "Practice again" CTA, and Discard sits beside two safe actions with only a thin red border to mark it destructive.

## Page Descriptions

**Home ([Home.jsx](src/pages/Home.jsx))** — dark page, centred Vite template border down both sides, "Confidence Detector — AI Presentation Coaching — Real-time Feedback" header, four 250-px cards in a row (🎥 Live Practice, 📁 Analyze Recording, 🎤 Speech Analyzer, 📚 Library), then a thin "Last 5 sessions" sparkline strip with a delta pill. No hero, no value prop sentence — the five-word subtitle is the entire pitch.

**Live Practice ([LiveSession.jsx](src/pages/LiveSession.jsx))** — opens on a "Set up your practice" form (topic dropdown + duration slider 1–10 min + body text), then a full-screen 3-2-1 countdown, then "Requesting camera and microphone access…" with a spinner, then the active screen: optional red connection-lost banner, optional red unsupported-language banner, optional grey backpressure indicator, optional blue calibrating indicator, blue topic banner, elapsed/target progress bar, REC dot + duration + hand-position emoji + "✓ Face detected" status bar, mirrored video on the left + 180-px confidence gauge on the right, six signal bars labelled in plain English, 1–3 emoji tip bullets, live transcript box, score-over-time canvas, then a wide red "Stop Session" button at the bottom.

**Live Audio Analyzer ([LiveAnalyzer.jsx](src/pages/LiveAnalyzer.jsx))** — same shell as Live Practice without video and without the face/gesture/calibrating chrome. **Currently broken**: clicking the start button throws a ReferenceError before reaching getUserMedia.

**Speech Analyzer ([Analyzer.jsx](src/pages/Analyzer.jsx))** — file picker + audio preview with optional multi-segment trim, OR a "Record Live (real-time scores)" button that mounts the Live Audio Analyzer. The dual-purpose page is poorly signposted: no header copy explains that "Speech Analyzer" is just "audio-only Live Practice".

**Upload ([Upload.jsx](src/pages/Upload.jsx))** — dashed dropzone, click-to-pick (no drag-drop wired up), then a video preview + filename + duration + a "Use full clip" checkbox + optional segment editor + Analyze/Pick-different buttons. Processing shows a spinner with three rotating status strings.

**Result ([Result.jsx](src/pages/Result.jsx) + [SessionReport.jsx](src/components/SessionReport.jsx))** — for uploads: title/topic/tag editor, a 220-px gauge, signal bars with `?` tooltips, a collapsed "How was this computed?" details panel, feedback tips, four small cards (Face/Speech/Pace/Duration), speech-summary stats, synced playback review, expandable face-timeline rows, comments thread, then *Back to Library / Share / Discard & re-take*. For sessions: a coloured grade letter + 160-px gauge + Peak/Lowest/Duration stats in a header row, then signal breakdown, breakdown panel, score graph, progress chart, filler breakdown, pace stats, **insights**, **action items**, transcript, then the same back/share/discard footer. Insights and action items — i.e. the coaching — are the eighth and ninth sections.

**Library ([History.jsx](src/pages/History.jsx))** — search box + sort dropdown + Filters toggle + Clear button, optional date/score/tag filter panel, then a 1- or 2-column grid of cards. Each card has an optional "Shared by" pill, the title, the topic, a row with date/kind tag/duration/score, optional tag chips, a full inline `<video>`/`<audio>` player auto-loading metadata, then *View Result →* and *Delete*. Empty state is "No sessions yet." or "No recordings match your filters."

**How it works ([HowItWorks.jsx](src/pages/HowItWorks.jsx))** — long-form explainer: TOC, weights table, one section per signal, glossary, FAQ. Reasonable, scannable.

**Login / Register ([Login.jsx](src/pages/Login.jsx), [Register.jsx](src/pages/Register.jsx))** — single 420-px column with name/email/password fields and a single primary button. Clean.

## Top 10 Fix Priority

1. **L-01** — *Live Audio Analyzer is currently broken.* `setLanguageWarning` doesn't exist; the page can't start a recording. One-line rename to `setUnsupportedLanguage`.
2. **G-01** — *Vite-template `index.css` is fighting the real stylesheet.* Removing the unused parts (or deleting the file and the import) eliminates the gratuitous `border-inline`, the centred default text, and the conflicting font sizes. Single quickest visual win.
3. **L-02** — *Live screen is overwhelming.* The most-used page should be the calmest. Decide which two of the eight live elements are primary (probably video + a single coaching tip) and demote the rest to a collapsible drawer.
4. **R-02** — *No "Practice again" CTA after a result.* The whole product loop is "record → see report → record again". The footer points the user back to Library, not back to /live.
5. **G-03** — *Buttons need a system.* Six button classes plus dozens of inline-styled buttons make the surface feel built by committee. A 3-tier system (primary/secondary/destructive) replaces every current variant.
6. **R-01** — *Coaching is buried.* Insights and action items belong directly under the gauge, not after filler counts and pace charts.
7. **L-04** — *Camera-denied has no friendly screen.* The PermissionScreen component is already written; wire it up.
8. **LIB-01** — *Failed sessions are invisible.* Without a "Failed" pill, users will keep opening rows that have no playable content.
9. **LIB-02** — *Library autoloads N video players on mount.* Replace with thumbnails or click-to-load — page weight scales with library size today.
10. **G-04** — *ShareModal won't close on Escape.* TimelineModal already does it correctly; copy that handler to ShareModal (and reuse it as a hook so the next modal added doesn't repeat the bug).
